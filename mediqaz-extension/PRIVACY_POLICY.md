# Privacy Policy — MediQaz AI Ассистент Врача

**Дата вступления в силу:** 4 мая 2026 года  
**Последнее обновление:** 4 мая 2026 года

---

## 1. Общие сведения

MediQaz — расширение Chrome для врачей Казахстана, которое записывает приём, транскрибирует речь, генерирует медицинскую карту и помогает заполнять формы МИС.

Мы серьёзно относимся к защите данных пациентов и врачей.

## 2. Какие данные мы собираем

### Данные, хранящиеся ТОЛЬКО ЛОКАЛЬНО на устройстве врача:
- **Аудиозаписи приёмов** — хранятся в IndexedDB браузера (максимум 3 последних)
- **Транскрипции** — текст приёма, хранится в chrome.storage.local
- **Сгенерированные медкарты** — JSON с секциями медкарты
- **API ключи** — Deepgram и Groq ключи, хранятся в chrome.storage.local
- **Настройки врача** — имя, специальность, язык, инструкции (chrome.storage.sync)

### Данные, передаваемые третьим сторонам:
- **Deepgram Inc.** — аудио-поток отправляется для транскрипции в реальном времени через WebSocket
- **Groq Inc.** — текст транскрипции отправляется для генерации медкарты через API

## 3. Какие данные мы НЕ собираем

- ❌ Мы **НЕ** собираем данные на собственных серверах
- ❌ Мы **НЕ** храним данные пациентов в облаке
- ❌ Мы **НЕ** передаём данные рекламодателям
- ❌ Мы **НЕ** используем аналитику или трекеры
- ❌ Мы **НЕ** имеем доступа к данным врача или пациента

## 4. Хранение и удаление данных

- Все данные хранятся **исключительно локально** на устройстве
- Аудиозаписи автоматически удаляются по принципу FIFO (максимум 3 записи)
- Врач может удалить все данные через Настройки → «Очистить данные»
- При удалении расширения все данные удаляются автоматически

## 5. Безопасность

- API ключи хранятся в зашифрованном chrome.storage.local
- Расширение не использует внешние CDN или скрипты — весь код локальный
- Content Security Policy (CSP) запрещает выполнение inline-скриптов
- Данные не кэшируются на серверах Deepgram/Groq после обработки

## 6. Разрешения расширения

| Разрешение | Для чего |
|---|---|
| `storage` | Хранение настроек и API ключей |
| `sidePanel` | Открытие боковой панели |
| `activeTab` | Доступ к активной вкладке для автозаполнения МИС |
| `tabs` | Получение URL вкладки для определения МИС |
| `scripting` | Инжекция content.js для заполнения форм |
| `alarms` | Keep-alive Service Worker |

## 7. Права субъекта данных

Врач имеет полное право:
- Просматривать все сохранённые данные
- Удалять любые данные в любой момент
- Экспортировать данные в PDF
- Отключить расширение без потери контроля

## 8. Соответствие законодательству

Расширение разработано в соответствии с:
- Законом РК «О персональных данных и их защите» (2013)
- Правилами обработки медицинских данных РК
- Рекомендациями GDPR (для международной совместимости)
